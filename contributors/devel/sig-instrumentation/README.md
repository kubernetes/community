# SIG-Instrumentation Agent Skills

This directory contains agent skills designed to help automate metrics and instrumentation review and coding inside the `kubernetes/kubernetes` repository.

## Important Guidelines & Disclaimers

> [!IMPORTANT]
> **Human Review Required:** All suggestions, checklists, and reports generated by these agent skills **must be reviewed and verified by a human** before being posted as comments on any pull request. These tools are strictly optional resources and do not replace actual human reviews.

*   **Completely Optional:** Using these agent skills is entirely optional. It is not mandated for any contributors or reviewers to use an agent.
*   **Manual Trigger Only:** These tools are manual and are **not** wired into any automated CI/CD flows or pre-submit tests right now.


## Available Skills

### 1. K8s Metrics Reviewer (`k8s-metrics-reviewer`)
*   **Path:** `contributors/devel/sig-instrumentation/k8s-metrics-reviewer`
*   **Purpose:** Reviews a pull request or diff containing metrics/instrumentation changes against SIG-Instrumentation standards.
*   **Trigger Phrases:**
    *   `"review this metrics PR"`
    *   `"sig-instrumentation review"`
    *   `"is this metric stable/named correctly"`
*   **Example Prompt:**
    > Review the metrics in this PR: https://github.com/kubernetes/kubernetes/pull/12345

### 2. K8s Metrics Instrumenter (`k8s-metrics-instrumenter`)
*   **Path:** `contributors/devel/sig-instrumentation/k8s-metrics-instrumenter`
*   **Purpose:** Helps write or update metrics instrumentation code following Kubernetes standards and the stability framework.
*   **Trigger Phrases:**
    *   `"write this metrics PR"`
*   **Example Prompt:**
    > Write a metrics PR to add a new stable counter named apiserver_reconciliation_total tracking apiserver sync actions.
