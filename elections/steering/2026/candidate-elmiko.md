## What I have done

I joined the Kubernetes community in late 2015 building machine learning and
data science applications, and teaching others to do the same. Around 2019,
I started contributing to the Kubernetes Cluster Autoscaler and Cluster API
projects, becoming the primary maintainer for the Cluster API integration with
the Autoscaler. After a few years of contributing to Cluster API and the
Autoscaler, I started to work more closely with cloud controller projects and
began contributing to the SIG Cloud Provider community eventually becoming a
chair and technical leader for the SIG.

As a member of the SIG Cluster Lifecycle community, I led the feature group
for the Karpenter Cluster API provider. We worked for a year to
research, design, and build the provider, which is now an official sub-project
of Kubernetes. In 2023, I was honored to receive a
[Kubernetes Community Award](https://www.kubernetes.dev/community/awards/2023/)
for my work on the Cluster API project.

## What I'll do

I see serving on the Steering Committee as one of the most serious opportunities
to collaborate on the health and future of the Kubernetes project and community.
I would like to leverage my passion for open source software development to
improve the experience for new contributors by ensuring they have the necessary
support to be successful in the Kubernetes community. Further, I believe that
testing and automation are essential tools for building reliable software and I
will bring my experience to aid in supporting the advancement of continuous
integration activities in Kubernetes.
