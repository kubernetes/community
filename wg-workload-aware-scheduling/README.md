# Workload-aware Scheduling Working Group

Establish native APIs and a common scheduling substrate in core Kubernetes to evolve beyond pod-by-pod placement and natively support workload-aware scheduling paradigms.

The [charter](charter.md) defines the scope and governance of the Workload-aware Scheduling Working Group.
