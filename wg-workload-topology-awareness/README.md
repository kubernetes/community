# Workload-Topology Awareness Working Group

The working group is dedicated to exploring and improving workload topology awareness in Kubernetes. This should result in better workload scheduling and placement,  better workload autoscaling, better workload migration and availability, load balancing, de/scheduling, and cloud provider and third-party integrations.

The [charter](charter.md) defines the scope and governance of the Workload-Topology Awareness Working Group.
