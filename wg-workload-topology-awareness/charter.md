# WG Workload-Topology Awareness Charter

This charter adheres to the conventions described in the [Kubernetes Charter README] and uses
the Roles and Organization Management outlined in [wg-governance].

## Scope

WG Workload-Topology Awareness is dedicated to exploring and improving workload topology awareness in Kubernetes. 
This should result in better workload scheduling and placement, better workload autoscaling, better workload migration 
and availability, load balancing, de/scheduling, and cloud provider and third-party integrations.

WG Workload-Topology Awareness [readme]

- Autoscaling related features (e.g. Integration with Cluster Autoscaler)
- Scheduling related features (e.g. gang-scheduling plugin)
